Obj basic

Object are use to store data and more complex entities. A properties is a "Key: value" pair, where the key is a string also called property name and value can be anything.Square brackets are mush more powerful than dot notation.

in:

You can use the in operator and it will help you if you have a properties in a obj.

let user = { name:"Juan", age:"30")

alert("age in user")// true, user exists

alert("blabla" in user) // false

"for...in" loop

for (key in object) {

// executes the body for each key among object properties

}

Object Reference

Objects are assigned and copied by reference. in other word, a variable stores not the "object value" but a "reference"(address in the hemp or memory) for the value. So copying such a variable or passing it as a function arguments copies the reference , not the object.

Garbage Collection

  • Garbage collection is performed automatically.
  • Object are retained in memory while they are reachable.
  • Being referenced is not the same as being reachable. A pack of interlinked objects can become unreachable a whole.

"this"

  • Methods can reference the object as this
  • When a function is declared, it may use this, but that has no value until the function is called.
  • A function can be copied between objects.
  • Arrow function are special: they have no this. When this is accessed inside an arrow function, is taken from the outside.
